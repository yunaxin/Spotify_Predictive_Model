# Spotify_Predictive_Model

## Introduction

This project analyzes the Spotify Music Tracks dataset, which contains 114,000 tracks across a wide range of genres. Each track includes metadata such as artist, release date, duration, explicit status, and genre, along with audio features generated by Spotify, including danceability, energy, valence, acousticness, instrumentalness, loudness, and tempo.

The goal of this project is to understand whether a track’s audio characteristics and genre can help predict its popularity. This question is relevant to real-world music analytics because platforms, artists, and playlist curators often rely on data-driven insights to understand listener behavior and identify patterns behind successful songs. By examining both genre-level trends and track-level audio features, this project explores how measurable properties of a song may relate to its popularity.

For the prediction task, a track is labeled as popular if its Spotify popularity score is at least 55. This creates a binary classification problem: predicting whether a track is popular or not popular using information that would be available about the song, such as its genre, audio features, duration, and explicit status.



Prediction Problem

'Can we predict whether a Spotify track will be popular (popularity ≥ 55) using its audio features, genre, and metadata?'

This is a **binary classification** problem because the response variable, `is_popular` can take only one of two possible values: 

`True(1)`: The track has a popularity score greater than or equal to 55.
`False(0)`: The track has a popularity score less than 55.
